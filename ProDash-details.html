Project Overview

ProDash is a centralised project management tool designed to streamline team collaboration, task management, and communication within one unified platform. The goal of ProDash is to reduce cognitive load by minimizing the need to switch between multiple apps, thereby improving team efficiency and productivity.

Discover
Problem Statement

Teams often face the challenge of managing projects across various tools, leading to fragmented workflows and frequent app-switching. This not only disrupts focus but also creates communication gaps and decreases productivity.

Problem Space

“App Switching”is sucking productivity, not creating it. Twenty-six percent (26%) of respondents say that one of their biggest challenges is switching from platform to platform across multiple platforms, with half of them adding that they would prefer one solution that could handle all of their different tasks.

- Forbes (March 2023)

Research

Objective:
The purpose of this research is to investigate the impact of using multiple collaboration tools on team productivity.
I aim to understand the challenges faced by employees when navigating through tools, as well as the consequences of app overload on workflow efficiency and creative processes.

The basic truth is that multiple apps are killing team productivity, making it increasingly difficult to find information, manage multiple UIs, breaking up creative flows ...

- Spike: Using Multiple Apps at Work Kills Team Productivity ... (February, 2023)

... workers are saddled with too many of these disconnected apps, leading to poor processes, increased errors, and wasted actions that could otherwise be automated

- PEGA: ... Employees Switch Apps Over 1,100 Times a Day (December 2018)

... workers switch between 10 apps 25 times per day and miss over one-quarter of deadlines.

- CIO: App switching is killing your team’s productivity ... (Apr 2021)

.. switching between different communication channels can leave staff feeling fragmented and disparate.

- Forbes: How Companies Can Prevent App Overload ... (February, 2023)
Assumptions:
  • Customers value time-saving features that automate repetitive tasks and streamline communication processes.
  • Risk of resistance to change among users who are accustomed to their current set of tools, hindering adoption rates.
  • Customers struggle with the inefficiency of managing multiple collaboration tools, leading to decreased productivity and increased frustration.
  • Customers seek a solution that provides seamless integration, simplified workflows, and enhanced usability with multi-platform connectivity.
Desired Outcomes:
1

Identify the key factors contributing to app overload and frequent switching between applications among employees.
2

Examine the extent to which app overload affects productivity, including its impact on information retrieval, task completion rates, and overall efficiency.
3

Assess the effectiveness of current strategies and tools employed by organisations to mitigate the negative effects of app overload.
4

Propose actionable recommendations and best practices for streamlining collaboration tools, optimising workflow processes, and enhancing team productivity

Epics & User Stories

From synthesising the interview findings and affinity mapping, I was able to identify three potential design solutions to the problem space with the following epics:


Unified Collaboration Platform
This directly addresses the core problem of fragmented workflows caused by switching between multiple tools by centralising all project-related tools such as:
  • communication
  • task management
  • document sharing
The unified platform will foster seamless collaboration, ensuring all relevant tools are in one place, streamlining team efforts and boosting productivity.

Streamlined Workflow Automation
This epic aims to simplify workflows, reduce time spent on administrative tasks, and allow teams to focus on more strategic activities.

Automating repetitive tasks such as
  • status updates
  • role assignments
  • smart notifications
This will help teams to work more efficiently and avoid manual errors.

Enhanced Communication & Transparency
Integration of real-time messaging and project updates into the platform, will provide a solution to the lack of clarity and miscommunication.

Transparent communication fosters accountability and collaboration, allowing teams to quickly address issues and keep everyone aligned with the project’s goals. Essentially promoting a collaborative work environment.
Task Flows

Dashboard Setup: The user can login and be directed to the dashboard screen. If they’re a new user, they will either need to register as a fresh user and follow the set-up process or use their invite code to join an existing team, which will automatically set-up the dashboard.

View Project: From the centralised dashboard, users can enter specific projects to assign and monitor tasks, view project’s documents and Kanban/Scrum Board. The user is prompted to create a project if no project is found upon entering the main Dashboard or when the user selects ‘View All Projects’.

Update Task Details & Communication: When viewing a task, the user is able to make changes to the task’s details such as status, deadlines, dependencies, documents, and more. Within the Communication Tab, the user is able to input any important information from meetings or view previous messages regarding issues etc. Upon saving the changes, the system collates all changes into a summarised message which is broadcasted to assigned team members.

User Testing

I conducted a round of user testing with 5 participants on the low-fidelity wireframes, to evaluate the usability and functionality of the proposed centralised dashboard. This is to ensure I'm desigining with the users in mind before starting high-fidelity wireframing. I presented the following scenario to the test participants:

Assuming you’re an existing user, please go ahead to login to view the dashboard. Navigate through the dashboard and view more projects/tasks. Please provide any thoughts you have on the layout of the pages, different tile designs, and navigating across the different pages

The changes between the two iterations of the low-fidelity wireframes were determined by the following issues found during user testing:


No Branding Elements
Many test users expressed that their first impression of the prototype was not pleasant as the were not presented with an introduction or landing page to introduce the app to them.

Being initially presented with the login page provides a negative user experience, which I didn’t take into consideration when designing the wireframes.

Call Button
Due to the assigned members icons were misunderstood, this caused for users to not fully understand the purpose of the call button.

Another potential issue identified is that the call button could be accidentally swiped when sliding the project tiles to the right.

Information Overload
Users felt like too much information was presented on the project and task tiles, therefore making the dashboard screen look too clustered.

They expressed they would rather have the dashboard to be an overview of projects and see more information on the project screen.

Incorrect Icons
  • The assigned members on the project and task tiles were incorrectly identified by users, which caused further confusion with the call button.
  • The tasks and issues icon that were used on the project tile were mistaken as reports and alerts icons, respectively.
  • The icon used to signify that a event tile is marked as busy was incorrectly identified as an cancelled/rejected event.

Copy Confusion
Users were not able to many any differentiation between the project and task tiles as the same copy was used for each one.

This was an issue as they were not able to a make connection to which tasks or event belonged to a specific project.

Prompt Message
Due to the same copy being used, users didn’t realise the task tile disappeared when they marked it as complete.

Many users expressed how they would’ve preferred to see a prompt message that a task has been marked instead of disappearing immediately.
